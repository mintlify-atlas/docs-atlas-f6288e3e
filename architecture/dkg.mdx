Distributed Key Generation (DKG) enables subnet nodes to collectively generate threshold signature keys without any single party knowing the complete secret key. This is essential for the Internet Computer's security architecture.

## Overview

DKG allows a group of nodes to:
- **Generate** threshold keys cooperatively
- **Distribute** secret shares to each participant
- **Avoid** trusted third parties or central dealers
- **Reshare** keys for membership changes or key rotation

**Location**: `rs/consensus/dkg/`

## Non-Interactive DKG (NI-DKG)

The Internet Computer uses a **non-interactive** DKG protocol based on Groth20:

**Key Properties**:
- **Non-interactive**: Dealers publish dealings independently
- **Consensus-based**: Dealings included in blockchain
- **Fault-tolerant**: Continues if enough honest dealers participate
- **Publicly verifiable**: Anyone can verify dealing correctness

## DKG Protocol Flow

### Phase 1: Configuration

Consensus creates DKG configuration:

```rust
pub struct NiDkgConfig {
    dkg_id: NiDkgId,
    dealers: BTreeSet<NodeId>,
    receivers: BTreeSet<NodeId>,
    threshold: NumberOfNodes,
    registry_version: RegistryVersion,
    resharing_transcript: Option<NiDkgTranscript>,
}
```

**Configuration Types**:
- **Initial DKG**: First time key generation (no resharing transcript)
- **Resharing DKG**: Key rotation (has resharing transcript)

### Phase 2: Dealing Creation

Each dealer creates and publishes a dealing:

```rust
fn create_dealing(
    &self,
    dkg_pool: &dyn DkgPool,
    config: &NiDkgConfig,
) -> Option<ChangeAction>
```

*Reference*: `rs/consensus/dkg/src/lib.rs:108-162`

**Process**:
1. Check if node is dealer for this config
2. Check if dealing already created
3. Load resharing transcript if needed
4. Call crypto to create dealing
5. Sign the dealing
6. Return action to add to validated pool

**Dealing Content**:
```rust

```

*Reference*: `rs/consensus/dkg/src/lib.rs:19, 264`

Nodes validate dealings from others:


**Validation Checks**:

   ```rust
       return Mutations::from(ChangeAction::RemoveFromUnvalidated(message));
   ```

2. **Height check**: Dealing for current DKG interval

   ```rust
       return Mutations::new();  // Skip
   ```

   *Reference*: `rs/consensus/dkg/src/lib.rs:202-205`

3. **Config check**: Config exists for this DKG ID
   ```rust

       Some(config) => config,
       None => return invalidate("No DKG configuration found"),
   ```

   *Reference*: `rs/consensus/dkg/src/lib.rs:209-218`

4. **Dealer check**: Signer is authorized dealer
   ```rust

       return Err(InvalidDealer(dealer));
   }
   ```
   *Reference*: `rs/consensus/dkg/src/lib.rs:267-269`

5. **Cryptographic verification**:
   ```rust
   crypto.verify(message, config.registry_version())?;
   NiDkgAlgorithm::verify_dealing(
       crypto,
       config,
       dealer,
       &message.content.dealing,
   )?;
   ```
   *Reference*: `rs/consensus/dkg/src/lib.rs:270-276`

### Phase 4: Dealing Aggregation

Consensus collects sufficient dealings:
- **Threshold**: Need `threshold` valid dealings
- **Redundancy**: Can use more than threshold
- **Inclusion**: Payload builder includes dealings in blocks

### Phase 5: Transcript Creation

Once enough dealings are finalized:
1. Consensus creates transcript from dealings
2. Transcript becomes part of summary block
3. Nodes load transcript via DkgKeyManager
4. Threshold signature operations become available

## DKG Types and Intervals

### DKG Tags

Different DKG instances for different purposes:

```rust
pub enum NiDkgTag {
    LowThreshold,    // f+1 threshold for notarization
    HighThreshold,   // 2f+1 threshold for certification
}
```

**Low Threshold**:
- Used for: Random beacon, notarization
- Threshold: f+1 (simple majority)
- More efficient, available with fewer nodes

**High Threshold**:
- Used for: Certification, finalization
- Threshold: 2f+1 (supermajority)
- Stronger security, requires more nodes

## Resharing Protocol

### Purpose

Resharing allows updating key shares while keeping public key constant:
- **Membership changes**: Add/remove nodes
- **Key rotation**: Improve security
- **Subnet recovery**: Restore lost shares

### Resharing Transcript

```rust
pub struct NiDkgConfig {
    // ...
    resharing_transcript: Option<NiDkgTranscript>,
}
```

When resharing:
1. **Old receivers** become dealers
2. **Resharing transcript** provides old key material
3. **New receivers** get fresh shares
4. **Public key** stays the same
5. **Old shares** become invalid

### Loading Transcripts

```rust

    && !self.dkg_key_manager
        .lock()
        .unwrap()
        .is_transcript_loaded(&transcript.dkg_id)
```

## Pool Management

### DKG Pool

Separate artifact pool for DKG dealings:

```rust
pub trait DkgPool {
    fn get_validated(&self) -> Box<dyn Iterator<Item = &Message>>;
    fn get_unvalidated(&self) -> Box<dyn Iterator<Item = &Message>>;
    fn get_current_start_height(&self) -> Height;
}
```

**Sections**:
- **Unvalidated**: Newly received dealings
- **Validated**: Verified dealings ready for inclusion

### Pool Mutations

```rust
pub enum ChangeAction {
    AddToValidated(Message),
    MoveToValidated(Message),
    RemoveFromUnvalidated(Message),
    HandleInvalid(DkgMessageId, String),
    Purge(Height),
}

pub type Mutations = Vec<ChangeAction>;
```

*Reference*: `rs/consensus/dkg/src/lib.rs:8-9` (interface definitions)

**Actions**:
- **AddToValidated**: New dealing from this node
- **MoveToValidated**: Validated received dealing
- **RemoveFromUnvalidated**: Duplicate or outdated dealing
- **HandleInvalid**: Mark dealing as invalid with reason
- **Purge**: Remove all dealings below height

### Purging

Keep pool bounded:

```rust
let dkg_summary_block = self.consensus_cache.summary_block();
let start_height = dkg_summary_block.height;

if start_height > dkg_pool.get_current_start_height() {
    return ChangeAction::Purge(start_height).into();
}
```

*Reference*: `rs/consensus/dkg/src/lib.rs:297-302`

When new DKG interval starts, purge old dealings.

## Bouncer (Gossip Control)

```rust
pub struct DkgBouncer {
    metrics: BouncerMetrics,
}

impl<Pool: DkgPool> BouncerFactory<DkgMessageId, Pool> for DkgBouncer {
    fn new_bouncer(&self, dkg_pool: &Pool) -> Bouncer<DkgMessageId> {
        let start_height = dkg_pool.get_current_start_height();
        Box::new(move |id| {
            match id.height.cmp(&start_height) {
                Ordering::Equal => BouncerValue::Wants,
                Ordering::Greater => BouncerValue::MaybeWantsLater,
                Ordering::Less => BouncerValue::Unwanted,
            }
        })
    }
}
```

*Reference*: `rs/consensus/dkg/src/lib.rs:352-390`

**Purpose**: Control which DKG artifacts to request via P2P
- **Wants**: Current interval dealings
- **MaybeWantsLater**: Future interval dealings
- **Unwanted**: Past interval dealings

## Dealing Uniqueness

Only one dealing per dealer per DKG:

```rust
if contains_dkg_messages(dkg_pool, config, *dealer_id) {
    return Mutations::from(ChangeAction::RemoveFromUnvalidated(message));
}
```

*Reference*: `rs/consensus/dkg/src/lib.rs:233-235`

**Rationale**:
- Dealing generation is non-deterministic
- Honest nodes may create multiple valid dealings
- Accept only first valid dealing from each dealer
- Remove duplicates from unvalidated pool
