## Overview

XNet (Cross-Network) messaging enables communication between canisters on different subnets of the Internet Computer. It implements a reliable, ordered message delivery protocol with cryptographic guarantees, allowing subnets to operate independently while maintaining global state consistency.

## Architecture

```
Subnet A                          Subnet B
+----------------+              +----------------+
| Canister       |              | Canister       |
|   Request  ----+--> Stream -->+---> Response   |
|   Response <---+--- Stream <--+---- Request    |
+----------------+              +----------------+
        |                              |
        v                              v
  State Manager                  State Manager
        |                              |
        v                              v
  XNet Payload                   XNet Payload
    Builder                        Builder
        |                              |
        +--------> Registry <----------+
```

### Key Concepts

**Stream**: Unidirectional, ordered message queue between two subnets

**Stream Slice**: Subset of stream messages with cryptographic proof

**XNet Payload**: Collection of stream slices included in a consensus block

**Certified Slice**: Stream slice with Merkle proof and threshold signature

## Stream Model

### Stream Structure

Each subnet pair has bidirectional streams:

### Message Ordering

**StreamIndex**: 64-bit monotonically increasing index

```rust
pub struct StreamIndexTag;
pub type StreamIndex = AmountOf<StreamIndexTag, u64>;

pub struct StreamIndexedQueue<T> {
    begin: StreamIndex,
    queue: VecDeque<T>,
}
```

**Guarantees**:
- Gap-free delivery (no missing indices)
- In-order processing
- Exactly-once semantics

### Signals

Reverse-direction acknowledgments:

```rust
pub struct RejectSignal {
    pub reason: RejectReason,
    pub index: StreamIndex,
}

pub enum RejectReason {
    CanisterMigrating = 1,
    CanisterNotFound = 2,
    CanisterStopped = 3,
    CanisterStopping = 4,
    QueueFull = 5,
    OutOfMemory = 6,
    Unknown = 7,
}
```

**Signal Types**:
- **Accept**: Implicit (default, no signal object)
- **Reject**: Explicit with reason code

**Purpose**:
- Enable garbage collection of delivered messages
- Provide reject responses for failed induction
- Prevent unbounded stream growth

### Stream Limits


**Enforcement**:
- Maximum signals in header: prevents one-sided traffic
- Message limit: prevents memory exhaustion

1. **Query State**: Get current replicated state at certified height

2. **Compute Expected Indices**: Calculate next expected message/signal for each stream

```rust

        message_index: signals_end,
        signal_index: messages_begin,
    }
}
```

3. **Garbage Collect Pool**: Remove already-inducted messages

```rust

```

4. **Take Valid Slices**: Extract slices from pool within limits

```rust

        subnet_id,

        Some(byte_limit),

        Ok(Some((slice, slice_bytes))) => {
            // Validate slice
            // Return if valid
        }
        _ => None,

```

5. **Prioritize Signals**: Add header-only slices first

```rust
// First pass: header-only slices (signals only)
for (subnet_id, begin) in shuffled_stream_positions.iter() {
    if let Some((header_only_slice, _, size_bytes)) =
        take_valid_slice(*subnet_id, begin.clone(), Some(0), bytes_left)

        stream_slices.insert(*subnet_id, header_only_slice);

}
```

6. **Add Messages**: Replace headers with message slices where possible

```rust

    let msg_limit = get_msg_limit(subnet_id, &state);
    if let Some((slice, message_count, slice_bytes)) =
        take_valid_slice(subnet_id, begin, msg_limit, bytes_left)
    {
        stream_slices.insert(subnet_id, slice);
    }
}
```

7. **Build Payload**: Collect into XNetPayload


**Validation Checks**:

- **Signature**: Threshold signature from source subnet
- **Stream bounds**: `begin <= message_index <= end`
- **Slice bounds**: Messages within stream bounds
- **Index continuity**: Messages start at expected index
- **Signal validity**: Monotonic, within limits
- **Message limits**: Respects backlog-based limits

### Message Limits

Backpressure mechanism:

```rust
pub fn get_msg_limit(subnet_id: SubnetId, state: &ReplicatedState) 
    -> Option<usize> {
    use SubnetType::*;
    match state.metadata.own_subnet_type {
        Application | VerifiedApplication | CloudEngine => None,
        
        System => {
            // Limit incoming messages based on outgoing stream size
            state.streams().get(&subnet_id)
                .map(|stream| stream.messages().len())
                .map(|len| SYSTEM_SUBNET_STREAM_MSG_LIMIT.saturating_sub(len))
        }
    }
}
```

Prevents request/response imbalance on NNS and system subnets.

### Signal Validation

```rust
const MAX_SIGNALS: usize = MAX_STREAM_MESSAGES;

fn validate_signals(
    signals_end: StreamIndex,
    reject_signals: &VecDeque<RejectSignal>,
    expected: StreamIndex,
    state: &ReplicatedState,
) -> SignalsValidationResult {
    // 1. Check signals_end in valid range
    if expected > signals_end || signals_end > self_messages_end {
        return Invalid;
    }
    
    // 2. Check signal count limit
    if signals_end.get() - signals_begin.get() > 2 * MAX_SIGNALS {
        return Invalid;
    }
    
    // 3. Verify strict ordering
    let mut next = signals_end;
    for signal in reject_signals.iter().rev() {
        if signal.index >= next {
            return Invalid;
        }
        next = signal.index;
    }
    
    Valid
}
```

## Certified Stream Slice

### Structure

```rust
pub struct CertifiedStreamSlice {
    // Serialized state tree containing stream data
    #[serde(with = "serde_bytes")]
    pub payload: Vec<u8>,
    
    // Merkle proof for root hash computation
    #[serde(with = "serde_bytes")]
    pub merkle_proof: Vec<u8>,
    
    // Threshold signature on root hash
    pub certification: Certification,
}
```

### Certification Process

1. **State Tree**: Stream data inserted into replicated state tree

2. **Merkle Proof**: Generated for stream subtree

3. **Root Hash**: Computed from payload + proof

4. **Threshold Signature**: BLS signature from source subnet

5. **Verification**: Destination subnet validates using source subnet's public key

## Performance Optimizations

### Random Shuffling

Fair slice selection:

```rust
let mut shuffled_stream_positions: Vec<_> = 
    stream_positions.clone().into_iter().collect();
self.random_shuffle(&mut shuffled_stream_positions);
```

Prevents starvation when byte limit is exceeded.

### Header-First Strategy

Prioritizes signals:

```rust
// Rationale:
// - Signals are orders of magnitude smaller than messages
// - Signals unblock reverse streams (garbage collection)
// - Reject signals are equivalent to reject responses
```

## Error Handling

### Slice Validation Results

```rust
enum SliceValidationResult {
    Valid {
        messages_end: StreamIndex,

        byte_size: usize,
```

### Error Recovery

**Invalid Slices**: Dropped from pool, logged

**Empty Slices**: Skipped (no new data)

**Fetch Errors**: Retried in next refill cycle

**Validation Errors**: Payload rejected, metrics updated

## Security Considerations

### Cryptographic Guarantees

- **Authenticity**: Threshold signatures prevent forgery
- **Integrity**: Merkle proofs ensure data hasn't been tampered
- **Non-repudiation**: Signed by source subnet's threshold key

### DOS Prevention

- **Message limits**: Prevents unbounded growth
- **Signal limits**: Prevents rejection flooding  
- **Byte limits**: Respects block size constraints
- **Validation timeouts**: Prevents slow validation attacks

### Dishonest Subnet Detection

```rust
// Reject slices with too many signals
if signals_end.get() - signals_begin.get() > 2 * MAX_SIGNALS {
    log!("Too old reject signal in stream from {}", subnet_id);
    return Invalid;
}
```
