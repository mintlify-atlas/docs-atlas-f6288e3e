The execution environment is responsible for executing canister smart contracts written in WebAssembly. It provides sandboxed, deterministic execution with resource metering and system API access.

## Overview

The execution environment consists of several interconnected components that work together to safely execute canister code:

- **Hypervisor**: WebAssembly runtime with system call interface
- **Scheduler**: Orchestrates canister execution rounds
- **Canister Manager**: Lifecycle management (create, install, upgrade)
- **Query Handler**: Non-replicated query execution
- **Ingress Filter**: Validates incoming messages

<Info>
All canister execution is deterministic and metered to ensure consistent state across all subnet nodes.
</Info>

## Core Components

### 1. Hypervisor

The hypervisor is the WebAssembly runtime that executes canister code:

<Accordion title="Hypervisor Responsibilities">
- **WebAssembly Execution**: Runs Wasm code in a sandboxed environment
- **System API**: Provides canister APIs (call, trap, time, etc.)
- **Resource Metering**: Tracks instruction count and memory usage
- **Compilation**: JIT compiles Wasm to native code
- **Page Management**: Handles canister heap memory with copy-on-write

### 2. Scheduler

The scheduler orchestrates execution rounds across all canisters:

<Accordion title="Scheduler Features">
- **Round-based Execution**: Executes canisters in deterministic rounds
- **Fair Scheduling**: Distributes compute capacity across canisters
- **Instruction Limits**: Enforces per-message and per-round limits
- **Heap Delta Rate Limiting**: Prevents memory growth attacks
- **Long-running Execution**: Supports DTS (Deterministic Time Slicing)

## Resource Metering

The execution environment enforces strict resource limits:

### Instruction Limits

- **Per-message limit**: Maximum instructions for a single message
- **Per-round limit**: Maximum instructions per execution round
- **Slice limit**: Instructions per DTS slice for long executions

### Memory Limits

- **Wasm memory**: 4GB max per canister (32-bit address space)
- **Stable memory**: 400GB max (48-bit address space)
- **Heap delta rate limiting**: Limits memory growth rate

### Execution Time

- **Deterministic Time Slicing**: Pauses and resumes long-running executions
- **Round duration**: Target duration per execution round
- **Timeout handling**: Detects and handles stuck executions

## Sandboxing and Security

The execution environment provides multiple layers of security:

<Accordion title="Process Isolation">
Canister execution runs in separate sandboxed processes:
- Prevents memory corruption attacks
- Limits resource access
- Enables safe multi-tenancy
</Accordion>

<Accordion title="WebAssembly Sandboxing">
Wasm provides memory safety and control flow integrity:
- Linear memory isolation
- No direct system calls
- Controlled imports/exports
</Accordion>

<Accordion title="System API Restrictions">
Canisters can only access the environment through controlled APIs:
- No filesystem access
- No network access (except HTTPS outcalls)
- No arbitrary code execution
</Accordion>

## Determinism Guarantees

All execution must be deterministic for consensus:

1. **No external randomness**: Pseudo-random bytes from VRF
2. **No wall clock time**: Time comes from consensus
3. **No floating point**: Disabled in Wasm execution
4. **Deterministic scheduling**: Round-robin execution order
5. **Metered execution**: Instruction counting ensures termination

## Performance Optimizations

<Accordion title="Copy-on-Write Memory">
Canister heap memory uses CoW for efficient cloning:
- State snapshots share unchanged pages
- Only modified pages are copied
- Reduces checkpoint creation overhead
</Accordion>

<Accordion title="WebAssembly Compilation Caching">
Compiled Wasm modules are cached:
- Avoids recompilation on every execution
- Persisted across checkpoints
- Warmed up during checkpoint loading
</Accordion>

<Accordion title="Query Caching">
Query results can be cached for identical requests:
- Reduces redundant execution
- Improves query response times
- Configurable per canister
</Accordion>
