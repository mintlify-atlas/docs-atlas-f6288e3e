The `rs/interfaces/` directory contains trait definitions that establish public APIs between replica components. These interfaces enable loose coupling and modular architecture.

## Design Philosophy

The interfaces crate follows these principles:

1. **Interface Segregation** - Each interface focuses on a single responsibility
2. **Dependency Inversion** - Components depend on abstractions, not implementations
3. **Clean Boundaries** - Clear separation between component interfaces
4. **Associated Types** - Avoid direct dependencies on implementation types

### Registry Invariants

The registry client interface guarantees:

1. **Fully Available Versions**: All keys are available at `get_latest_version()`
2. **Known Zero State**: Registry version 0 is always empty
3. **Deterministic Queries**: Repeated queries return identical results

## Associated Types Pattern

Many interfaces use associated types to avoid coupling:

```rust
pub trait Pool {
    type Message;
    type Artifact;
    
    fn get(&self, id: &MessageId) -> Option<Self::Message>;
}
```

This approach is documented in the source as:

> "Several traits in this package use associated types to avoid direct dependency on ReplicatedState. This results in cleaner dependency graph and speeds up incremental compilation."
