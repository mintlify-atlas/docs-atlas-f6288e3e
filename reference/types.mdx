The `rs/types/` directory contains fundamental type definitions used throughout the IC replica. These crates provide the data structures that represent protocol entities, messages, and system state.

## Core Types Crates

### ic-base-types

Provides fundamental identifier and quantity types used across the entire codebase.

**Location**: `rs/types/base_types/`

**Key Types**:

```rust
// Identifier types using phantom types
pub type NodeId = Id<NodeTag, PrincipalId>;
pub type SubnetId = Id<SubnetTag, PrincipalId>;
pub type CanisterId = /* ... */;
pub type PrincipalId = /* ... */;

// Quantity types
pub type RegistryVersion = AmountOf<RegistryVersionTag, u64>;
pub type NumBytes = AmountOf<NumBytesTag, u64>;
pub type NumSeconds = AmountOf<NumSecondsTag, u64>;
pub type NumOsPages = AmountOf<NumOsPagesTag, u64>;
```

**Features**:
- Type-safe identifiers using phantom types from `phantom_newtype`
- Principal ID parsing and validation
- Canister ID conversion and serialization
- Formatted display for byte quantities (e.g., "123.45 MiB")
- Protobuf conversion support

**Dependencies**:
- `phantom_newtype` - Zero-cost wrapper types
- `ic-crypto-sha2` - Hashing utilities
- `candid` - Candid serialization
- `serde` - General serialization

## Type Design Patterns

### Phantom Types

Many ID types use phantom type parameters for type safety:

```rust
// Can't accidentally use a NodeId as a SubnetId
let node_id: NodeId = /* ... */;
let subnet_id: SubnetId = /* ... */;
// node_id == subnet_id; // Compile error!
```

### Newtype Pattern

Quantity types wrap primitives:

```rust
pub type NumBytes = AmountOf<NumBytesTag, u64>;

let bytes = NumBytes::new(1024);
let value: u64 = bytes.get();
```

### Protobuf Integration

Most types support conversion to/from protobuf:

```rust
impl From<Type> for pb::Type { /* ... */ }
impl TryFrom<pb::Type> for Type { /* ... */ }
```
